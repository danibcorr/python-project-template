## Project Structure

This project follows a modular and standardized layout to facilitate scalability,
collaboration, and maintainability:

```
├── .github/                <- GitHub Actions workflows and reusable CI/CD actions.
├── .gitignore              <- Specifies files and directories excluded from version control.
├── .pre-commit-config.yaml <- Configuration for pre-commit hooks.
├── .vscode/                <- VS Code workspace settings (optional).
├── LICENSE                 <- License file specifying the terms of use and distribution.
├── Makefile                <- Automation commands for tasks like installation, testing, and documentation.
├── zensical.toml           <- Configuration for building and deploying documentation using Zensical.
├── pyproject.toml          <- Central configuration file for dependencies, tooling (ruff, mypy), and metadata.
├── README.md               <- Project overview, usage instructions, and reference (this file).
├── uv.lock                 <- Lockfile generated by `uv`, ensuring consistent dependency resolution.
│
├── config/                 <- Project configuration files (e.g. JSON settings).
│
├── docs/                   <- Project documentation source for Zensical.
│
├── notebooks/              <- Jupyter notebooks for exploratory analysis or experiments (optional).
│
├── prompts/                <- Prompt files for AI-assisted workflows (optional).
│
├── reports/                <- Output reports, visualizations, and logs.
│   ├── imgs/               <- Auto-generated figures, charts, or plots used in reports.
│   └── logs/               <- Log files generated during data processing or training.
│
├── tests/                  <- Unit and integration tests for the project modules.
│
└── {{ cookiecutter.project_module_name }}/     <- Core source code of the project.
    ├── __init__.py          <- Marks the module as a Python package.
    ├── config/              <- Centralized configuration variables and constants.
    │   ├── __init__.py
    │   └── config.py
    ├── etl/                 <- Code related to data extraction, transformation, and loading (ETL).
    │   ├── __init__.py
    │   ├── extract.py
    │   ├── transform.py
    │   └── load.py
    ├── models/              <- Training, inference, and model-related logic.
    │   ├── __init__.py
    │   ├── train.py         <- Scripts and functions to train machine learning models.
    │   └── inference.py     <- Scripts and functions for model inference.
    └── utils/               <- Utility scripts and helper functions used across modules.
        ├── __init__.py
        └── utils.py
```
