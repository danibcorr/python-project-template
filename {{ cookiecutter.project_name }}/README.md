## Project Organization

```
├── .gitignore           <- List of files and folders ignored by Git
├── .python-version      <- Specifies the required Python version for the project
├── LICENSE              <- Open-source license (if one is chosen)
├── Makefile             <- Convenience commands like `make data` or `make train`
├── mkdocs.yml           <- MkDocs configuration for documentation
├── uv.lock              <- Lockfile generated by uv, contains pinned dependencies
├── README.md            <- This file; project overview for developers
│
├── docs/                <- MkDocs documentation project; see www.mkdocs.org for details
│
├── notebooks/           <- Jupyter notebooks. Naming convention: number + initials +
│                           short description, e.g. `1.0-jqp-initial-data-exploration`
│
├── pyproject.toml       <- Project configuration and metadata for src, includes settings
│                           for tools like Black, isort, etc.
│
├── reports/             <- Generated analysis in HTML, PDF, LaTeX, etc.
│   ├── figures/         <- Generated graphics and figures for reporting
│   └── logs/            <- Logs generated during project execution
│
├── requirements.txt     <- Requirements file for reproducing the analysis environment
│                           (e.g. generated with `pip freeze`)
│
├── setup.cfg            <- Configuration file for tools like flake8
│
└── src/                 <- Source code for use in this project
    ├── __init__.py      <- Makes `src` a Python module
    ├── config.py        <- Stores useful configuration variables
    ├── etl/             <- Code for Extract, Transform, Load (ETL) processes
    ├── models/          <- Trained models, model predictions, or summaries
    │   ├── __init__.py
    │   ├── predict.py   <- Code to run model inference
    │   └── train.py     <- Code to train models
    └── utils/           <- Utility functions and helpers
        ├── __init__.py
        └── utils.py
```
